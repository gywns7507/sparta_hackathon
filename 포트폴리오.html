문제를 어떻게 이해했나

문제를 되먹이는 것이 아니라, 내 언어로 다시 세우기

과제는 특정 기업에 종속되지 않은 마케터 직무 공통 시나리오입니다. 인스타그램·블로그·유튜브에 카드뉴스·숏폼·블로그·인포그래픽을 발행하고, CTR과 인게이지먼트율로 성과를 측정하는 환경을 가정합니다. 이 안에서 제가 진짜 체크포인트로 삼은 지점은 다음 세 가지였습니다.

  • 벤치마킹은 항상 수동이다. 발행 전마다 "예전에 비슷한 거 뭐 있었지"를 사람이 직접 뒤져야 함.
  • 발행 전 검증 수단이 없다. A/B 테스트는 발행 후 비교라, 저성과가 예상되는 콘텐츠를 걸러낼 방법이 없음.
  • 근거가 매번 감(感)이다. 시간대·이모지·포맷 선택이 데이터 없이 담당자 경험으로 반복됨.
이번에 의도적으로 풀지 않은 것 — 과거 성과 데이터는 70건뿐입니다. 이 규모에서 학습 기반 예측 모델을 만들면 과적합 위험이 훨씬 큽니다. 그래서 "예측 모델"이 아니라 "유사 콘텐츠의 실제 과거 성과로 기대 범위를 제시하는 패턴 진단"으로 문제 자체를 좁혔습니다. 데이터 규모에 맞는 방법을 고르는 것도 판단의 일부라고 생각했습니다.

어떻게 풀었나 — 접근과 판단

결과보다 재현 가능한 사고 과정

핵심 가설 — "형식과 채널이 같고, 시간대·이모지까지 비슷한 과거 콘텐츠의 실제 성과가 감보다 훨씬 나은 기준선이 된다."

진단 절차 (사람이 실제로 따라가는 순서)

  1. 형식·채널이 같은 과거 콘텐츠만 후보로 좁힌다

    카드뉴스와 숏폼처럼 소비 방식 자체가 다른 콘텐츠는 처음부터 비교 대상에서 제외합니다.

  2. 그 안에서 발행 시간대·이모지 여부까지 맞는 콘텐츠를 우선 정렬한다

    두 요소는 실제로 CTR 차이가 검증된 축이라, 맞을수록 더 신뢰도 높은 비교 대상으로 취급합니다.

  3. 가장 비슷한 3건의 실제 성과로 예상 CTR 범위를 제시한다

    추정이 아니라 실제로 발행됐던 콘텐츠의 실측치를 그대로 보여줍니다.

  4. 과거 데이터에서 실제로 차이 났던 요소로 개선안 2가지를 만든다

    표본이 작거나 근거가 약하면 "참고용"이라고 리포트에 그대로 표시합니다.

  5. (옵션) 팀 전체 발행 습관까지 훑어 구조적 문제를 우선순위와 함께 제안한다

    개별 콘텐츠 진단과 별개로, 반복되는 팀 차원 문제를 영향력 × 실행 난이도로 줄 세웁니다.

진행하며 내린 판단들 — 무엇을 버렸고 왜

기각
주제 카테고리를 유사도 기준에서 뺐다 처음엔 넣었지만, 검증해보니 성과와 통계적으로 무관하다는 게 확인돼 점수에서 제외했습니다. 근거가 무너지면 이미 짜둔 채점 기준도 스스로 고쳤습니다.
기각
조건을 끝까지 완화하는 방식을 버렸다 일부러 극단적인 가짜 데이터로 스트레스 테스트를 해봤더니, 후보가 부족하다고 조건을 계속 풀면 블로그와 숏폼처럼 완전히 다른 형식이 섞여 비교되는 오류가 났습니다. 형식·채널만은 끝까지 고정하기로 원칙을 바꿨습니다.
채택
팀 전략 우선순위는 곱셈으로 계산했다 영향력이 커도 실행이 극히 어렵다면 지금 당장 순위는 낮아야 한다고 봤습니다. 덧셈은 한쪽이 낮아도 다른 쪽으로 상쇄돼 우선순위를 부풀릴 위험이 있어 곱셈을 택했습니다.

AI와 직접 판단의 분업 — 반복적인 후보 비교·수치 계산·문서 재작성은 자동화에 맡겼고, "무엇을 비교 기준으로 인정할지" "근거가 통계적으로 실재하는지" "우선순위를 어떤 규칙으로 매길지" 같은 판단은 전부 직접 세우고 직접 검증했습니다.

낙관보다 정직한 스코프

적용에 필요한 것 — 발행 콘텐츠의 형식·채널·발행 시간·이모지 여부 같은 메타데이터와, CTR·인게이지먼트 실적 데이터만 있으면 동일한 구조로 재현할 수 있습니다. 특정 CMS나 툴에 묶여있지 않습니다.

스스로 인정하는 한계

  • 과거 데이터가 69건뿐이라 일부 비교(소표본 축)는 확정적 근거로 보지 않고 참고용으로만 표기했습니다.
  • 학습 기반 예측 모델은 이 데이터 규모에서 과적합 위험이 커서 의도적으로 배제했습니다.
  • 제목 길이 같은 일부 업계 통설은 자사 데이터로 검증할 사례 자체가 없어(n=0), 개선 제안의 근거로 쓰지 않고 참고 정보로만 남겼습니다.

2주가 더 주어진다면 — 실제로 발행한 뒤의 실측 CTR을 다시 입력해, 이 진단이 제시한 예상 범위와 실제 결과의 오차를 스스로 검증하는 피드백 루프를 추가하고 싶습니다. 데이터가 쌓이면 지금 참고용으로만 남긴 소표본 항목들도 다시 검증할 수 있습니다.

7

이 문제를 계속 맡는다면

발행 전 진단과 발행 후 재검증이 맞물리는 상시 루틴까지 확장할 수 있다고 봅니다. 지금은 한 번의 진단 절차지만, 데이터가 쌓일수록 같은 구조 위에서 근거가 더 촘촘해지는 방식으로 만들었습니다.

이번 결과물에서 제가 실제로 일한 방식 — 가설을 세우고, 직접 통계로 검증하고, 근거가 무너지면 이미 짜둔 채점 기준도 스스로 고치는 것 — 은 이 문서 전체에 그대로 기록돼 있습니다.

기술 상세 — 알고리즘·검증 방법 (기술 리뷰어용)

작업 절차

데이터 파악(결측·중복·성과 패턴 확인) → 유사도 기준 설계 → 리포트 생성 → 제출 전 자가 점검, 총 4단계로 진행했고 각 단계의 판단 근거는 전부 별도 로그로 기록했습니다.

유사도 알고리즘

앵커(필수, 완화하지 않음): 형식(type) + 채널(channel) 동시 일치. 서브 점수(앵커 안에서 정렬용, 3점 만점): 발행 시간대 일치 +2점, 이모지 일치 +1점. 동점 시 content_id 오름차순으로 결정론적 정렬해 재현성을 확보했습니다. 후보가 3건 미만이면 채널만 완화하고, 형식은 원칙적으로 끝까지 완화하지 않습니다.

통계 검증

일원배치 분산분석(ANOVA)과 Welch t-검정으로 각 축이 CTR과 실제로 유의미한 관계가 있는지 확인했습니다. 발행 시간대(η²=0.48)·이모지(Welch t=6.12)·형식·채널은 모두 유의했고, 주제 카테고리(η²=0.006)는 유의하지 않아 서브 점수에서 제외했습니다. CTR과 인게이지먼트율의 상관관계도 별도로 확인했습니다(r=0.89).

재현성 검증

다른 신규 콘텐츠 목록으로도 동일 절차가 재현되는지 확인했고, 특정 형식이 2건뿐인 극단적인 가짜 데이터로 스트레스 테스트를 진행해 조건 완화 로직의 버그(형식이 다른 콘텐츠가 섞여 비교되는 문제)를 발견·수정했습니다.

전략 우선순위 로직

우선순위 점수 = 임팩트(1~5) × 실행 난이도(1~5), 25점 만점. 해당 축이 통계적으로 유의하지 않으면 효과 크기와 무관하게 임팩트를 최저점으로 고정하도록 규칙화해, 근거가 약한 후보가 자동으로 후순위로 밀리게 만들었습니다.
