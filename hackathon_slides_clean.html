PROBLEM DEFINITION

문제 이해

기업 원문 문제

신규 콘텐츠 정보를 입력받아 과거 성과 데이터(70건) 패턴 근거로 ①유사 콘텐츠 TOP3 ②예상 CTR 범위·개선 제안 ③리포트를 산출. "ML 예측 모델이 아닌 패턴 기반 진단"이라는 제약 명시.

내가 재정의한 문제

진짜 문제는 예측력 부족이 아니라 "근거 있는 판단"과 "감"을 구분할 기준의 부재. 70건 소표본에서 진짜 영향 변수와 그럴듯해 보이는 무의미한 변수를 가려야, 개선 제안이 또 다른 감이 되지 않는다.

문제의 크기 (Before)

발행 전 검증 수단 없이 감으로 판단 → 유사 콘텐츠 벤치마킹도 매번 수동 반복 → 사람마다 다른 TOP3가 나와 재현이 안 됨.

스코프 — 푼 것 / 안 푼 것

O 5개 변수 통계 검증 + 유사도 매칭을 코드로 고정해 재현 가능하게 함
X ML 예측 모델 구축은 아님 (미션 제약대로 패턴 기반 진단에 한정)

04 / 09
APPROACH & DECISION

해결 접근 & 의사결정

접근 가설 / 핵심 흐름

/analyze(데이터 품질·기초 패턴) → /insight(유사도 기준 설계) → /generate(리포트 생성) → /review(자가 점검) 4단계를 거쳐, 전 과정을 scripts/pipeline.py 하나로 통합

버린 대안 & 이유

유사도 가중치를 감으로 정한 초안(시간대+2·주제+1·이모지+1)을 버리고 ANOVA·t-test·상관분석으로 5개 변수 전부 검증. topic_category는 η²=0.006으로 CTR과 무관해 가중치를 1→0으로 낮춤

▸ 막히는 지점 & 대화 방법

① 가중치를 감으로 정했다는 의심 → 통계 검증으로 방향 전환
② 69건에선 안 보이던 버그 → 극단 더미 데이터로 스트레스 테스트해 콘텐츠 유형 혼합 매칭 버그 발견
③ 배포 중 연쇄 오류(자동번역·pandas 충돌·한글 경로·Python 버전) → 로그를 그대로 공유하며 원인을 하나씩 좁힘

AI vs 내 판단

AI: 통계 분석 실행, 진단 리포트 작성, 배포 오류 원인 분석
나: 문제 재정의, 가중치 재설계 방향, 극단 케이스 검증 필요성, 배포 구조 수정 방향 결정

데이터 처리 과정

과거 콘텐츠 70건 CSV → 5개 변수 통계 검증(유의성·효과크기) → 가중치 재설계(topic_category η²=0.006으로 가중치 0) → 유사도 매칭 → CTR 범위 진단 → md 리포트 + 대시보드 출력
